后端知识学习-sql

简介

数据库作为一种专门管理数据的软件而出现。应用程序不需要自己管理数据，而是通过数据库软件提供的接口来读写数据。
关系数据库的关系模型是基于数学理论建立的。我们把域定义为一组具有相同数据类型的值的集合，给定一组域D1，D2，D3…Dn，它们的笛卡尔集定义为D1✖️D2…Dn。。。
数据类型
关系数据库支持的标准数据类型包括数值、字符串、时间等。
SQL
sql是结构化查询语言的缩写，用来访问和操作数据库系统。
SQL定义了以下几种操作数据库等能力：

  • DDL：data definition language，允许用户定义数据，也就是创建表、删除表、修改表结构；通常由数据库管理员执行。
  • DML：data manipulation language，允许用户添加、删除、更新数据，是应用程序对数据库等日常操作。
  • DQL：data query language，允许用户查询数据，是通常最频繁的数据库日常操作。 语法特点
    SQL的语言关键字不区分大小写。

关系模型

表的每一行称为记录（Record），记录逻辑意义上的一个数据。
表的每一列称为字段（column），同一个表的每一行记录拥有相同的columns。
字段定义了数据类型，以及是否允许为NULL。
同excel不同，关系数据库的表存在“一对多”，”多对一“，”一对一“的关系，需要按照应用程序的逻辑来组织和存储数据。

主键

对于关系表，任意两条记录不能重复，即不能完全相同，能够通过某个字段唯一区分不同的记录，这个字段称为主键key。
对于主键的要求，记录一旦插入到表中，主键最好不要再修改。主键是用来唯一定位记录的。
选取主键的基本原则：不实用任何业务相关的字段作为主键。
一般主键使用完全业务无关的字段，一般命名为id。常见的可作id的字段类型有：

  1. 自增整数类型
  2. 全局唯一GUID类型：使用一种全局唯一的字符串作为主键
    联合主键：两个或更多的字段都设置为主键，这种主键称为联合主键。对于联合主键，允许一列有重复，只要不是所有主键列都重复即可。

事务

在执行SQL语句的时候，某些业务要求，一些操作必须全部执行，而不能执行一部分，例如银行转账操作。如果有一条语句执行失败，则所有语句必须全部撤销。这种把多条语句作为一个整体进行操作的的功能，成为数据库的事务。如果事务失败，则效果和没有执行这些SQL语句一样，不会对数据库有任何改动。
数据库事务的ACID特性：

  • A：atomics，原子性，所有SQL作为原子工作但愿，要么全部执行，要么全部不执行。
  • C：consistent，一致性，事务完成后，所有数据状态是一致的，即A账户只要减去了100，B账户必定加上了100.
  • I：isolation，隔离性，如果有多个事务并发执行，每个事务作出的修改必须与其他事务隔离。
  • D：Duration，持久性，即事务完成后，对数据库数据的修改被持久化存储。
    对于单条SQL语句，数据库系统自动将其作为一个事务进行执行，成为隐式事务。
